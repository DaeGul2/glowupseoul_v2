# Glow Up Seoul — 운영자(어드민) 사용 설명서

> 이 문서 하나만 읽으면 처음 보는 사람도 어드민 페이지와 홈페이지를 운영할 수 있도록 만들었습니다.
> 컴퓨터 전문 용어는 최대한 뺐습니다. 모르는 단어가 나오면 그 자리에서 풀어서 설명합니다.
> 최종 작성: 2026-06-06

---

## 0. 가장 먼저 알아야 할 것 (꼭 읽으세요)

### 0-1. 어드민이 뭔가요
- **홈페이지** = 외국 환자가 보는 화면 (영어/중국어).
- **어드민 페이지** = 운영자(여러분)만 들어가는 관리 화면 (한국어). 주소는 `우리사이트주소/admin`.
- 어드민에서 입력한 내용이 곧 홈페이지에 나타납니다. **어드민이 원본, 홈페이지가 결과물**입니다.

### 0-2. 로그인
1. `/admin` 으로 들어갑니다.
2. **관리자 키**(비밀번호 같은 것)를 입력합니다. 운영 책임자에게 받으세요.
3. 들어가면 왼쪽에 메뉴, 가운데에 내용이 보입니다.

### 0-3. 왼쪽 메뉴 구조 (5개 묶음)
메뉴는 성격별로 묶여 있습니다. 위에서부터 순서대로 채우면 됩니다.

| 묶음 | 메뉴 | 한 줄 설명 |
|---|---|---|
| **카탈로그 (기본 재료)** | 카테고리 · 시술 · 기기 · 고민 | 병원과 상관없는 "정보 그 자체". 모든 것의 바탕. |
| **매트릭스 (연결)** | 고민↔시술 · 시술↔기기 | 위 재료들을 서로 연결. **자동 추천 품질의 80%가 여기서 결정됩니다.** |
| **병원·인력** | 병원 · 의사 · B&A(전후사진) | 실제 병원, 의사, 전후 사진. 외국 환자가 신뢰하는 정보. |
| **콘텐츠** | 실시간 피드 | 메인 화면에 흐르는 "M. in Singapore — HIFU" 같은 사회적 증거. |
| **운영·모니터링** | AI 스캔 내역 | 비용/호출 기록 보기 전용. 입력하지 않습니다. |

> 참고: **"브랜드"와 "병원×시술 가격표"는 따로 메뉴가 없습니다.** 너무 복잡해서 숨겼습니다. → "브랜드"는 병원 입력 화면 안에 **"브랜드 정보"** 칸으로 들어있고, "병원×시술 가격표"는 병원 입력 화면 아래 **"이 병원이 제공하는 시술"** 패널에서 관리합니다. (자세히는 §10 병원 편)

### 0-4. 모든 화면에 공통으로 적용되는 규칙 (★중요)

1. **별표(*)가 붙은 칸 = 반드시 입력.** 비우면 저장이 안 됩니다.
2. **"URL 식별자"** 라는 칸이 자주 나옵니다.
   - 영어 소문자 + 숫자 + `_`(밑줄)로 된 짧은 이름입니다. 예: `hifu_face`, `hershe_청담점`.
   - 이게 인터넷 주소, 사진 저장 폴더, 자동 연결에 전부 쓰입니다.
   - **한 번 정하면 절대 바꾸지 마세요.** 바꾸면 외부에 공유된 링크와 사진이 전부 깨집니다.
   - **같은 메뉴 안에서 중복될 수 없습니다.** (예: 시술 두 개가 똑같은 `hifu_face`를 쓰면 안 됨)
3. **사진 칸**은 회색 박스에 끌어다 놓거나 클릭해서 올립니다. 올리면 자동으로 저장되고 주소가 채워집니다. URL을 손으로 칠 필요 없습니다.
4. **다국어 칸** (한국어/영어/중국어/일본어): 한국어와 영어는 가능한 다 채우세요. 중국어/일본어는 비워도 되지만, 중국 환자 비중이 크니 중국어도 채우면 좋습니다.
5. **"활성 (사이트 노출)"** 체크박스: 체크를 풀면 그 항목이 홈페이지 어디에도 안 보입니다. 잠깐 숨길 때 사용. 보통은 체크된 채로 둡니다.
6. 입력 후 **저장** 버튼. "저장됨" 메시지가 뜨면 성공.
7. 삭제는 완전 삭제가 아니라 **숨김 처리**(복구 가능)입니다. 실수해도 괜찮습니다.

### 0-5. ? 아이콘
각 칸 라벨 옆 작은 동그란 **?** 에 마우스를 올리면 그 칸 설명이 뜹니다. 이 문서와 같은 내용입니다.

### 0-6. "이건 홈페이지 어디에 보이나요?" 표기 읽는 법
이 문서의 각 필드마다 **[노출 위치]** 를 적어놨습니다.
- **🟢 환자에게 보임** = 홈페이지 화면에 직접 나옵니다.
- **🟡 추천에만 영향** = 화면엔 안 보이지만, 스캔 후 자동 추천 결과(어떤 병원이 위에 오는지)에 영향을 줍니다.
- **⚪ 내부용** = 운영자만 보는 정보. 환자 화면엔 안 나옵니다. (그래도 정리해두면 나중에 유용)

## 1. 어떤 순서로 입력해야 하나요 (전체 흐름)

뒤 항목이 앞 항목을 필요로 하므로 **반드시 이 순서**로 채웁니다.

```
1) 카테고리        (이미 8개 들어있음 — 사진만 추가)
2) 고민            (검색 키워드)
3) 시술            (HIFU 리프팅 등)
4) 기기            (울쎄라 등)
5) 고민↔시술 매트릭스   ← 추천 품질의 핵심
6) 시술↔기기 매트릭스
7) 병원            (+ 브랜드 정보 + 제공 시술 + 의사 + 전후사진 한 화면에서)
8) 실시간 피드      (메인 화면 부팅용 5~10개)
```

> 처음 비어있는 환경이라면 병원 1곳당 약 30분. 자세한 시간 추정과 의존 관계는 `docs/data_seeding_order.xlsx` 참조.

## 2. 카테고리 (메뉴: 카탈로그 > 카테고리)

**무엇인가:** 시술과 고민을 묶는 가장 큰 분류. 현재 8개(얼굴/눈/코/바디/피부/헤어/웰니스/치과)가 이미 들어있습니다. **새로 만들기보다 사진만 채우는 일이 대부분**입니다.

| 칸 이름 | 무엇을 넣나 / 예시 | 필수 | 중복금지 | 노출 위치 |
|---|---|---|---|---|
| URL 식별자 | `face`, `eyes`, `nose` 같은 영문 키 | ★ | ✔ | 🟢 카테고리 페이지 주소 (`/category/face`) |
| 카테고리명 (한국어) | `얼굴`, `눈`, `코` | ★ |  | 🟢 홈 카테고리 카드 + 카테고리 페이지 제목 |
| 카테고리명 (영어) | `Face`, `Eyes`, `Nose` | ★ |  | 🟢 (영문 환자가 보는 카드 제목) |
| 카테고리명 (중국어) | `面部`, `眼部` |  |  | ⚪ 현재 화면 미표기 (채워두면 좋음) |
| 카테고리명 (일본어) | `顔` |  |  | ⚪ 현재 화면 미표기 |
| 도메인 | 목록에서 선택 (얼굴 미용/바디 윤곽/재생/수술/피부과/치과) | ★ |  | ⚪ 내부 분류용 |
| 상위 카테고리 | 보통 **비워둡니다**. 하위 카테고리를 만들 때만. |  |  | ⚪ 내부용 |
| 노출 순서 | 낮은 숫자가 먼저. 예: 10, 20, 30 |  |  | 🟢 홈에서 카드 나열 순서 |
| 카드 썸네일 | 정사각/4:3 사진 | | | ⚪ 현재 카드는 아래 '히어로'를 씁니다 |
| **카테고리 페이지 히어로** | 가로로 넓은 큰 사진 (1600px 이상) | | | 🟢 **홈 카테고리 카드의 배경 사진** + 카테고리 페이지 상단 |
| 활성 (사이트 노출) | 보통 체크 | | | 체크 풀면 홈에서 사라짐 |

> 팁: 홈 카테고리 카드의 배경 사진은 **"카테고리 페이지 히어로"** 칸을 씁니다. 비어 있으면 카드가 밋밋해지니 8개 모두 채우세요.

| 고민 (한국어) | `처짐`, `모공`, `여드름 흉터` | ★ |  | 🟢 카테고리 페이지 필터 칩 + 스캔 화면 |
| 고민 (영어) | `sagging`, `pores`, `acne scars` | ★ |  | 🟢 (영문 환자가 보는 칩) |
| 고민 (중국어) | `松弛`, `毛孔` |  |  | ⚪ 현재 화면 미표기 |
| 고민 (일본어) | |  |  | ⚪ 현재 화면 미표기 |
| 설명 (한국어) | (선택) 이 고민에 대한 짧은 설명 |  |  | ⚪ 현재 화면 미표기 |
| 설명 (영어) | |  |  | ⚪ 현재 화면 미표기 |

## 16. (꼭 이해하세요) 스캔하면 왜 이 병원이 추천되나요

운영자가 매트릭스를 잘 채워야 하는 이유입니다. 추천은 두 단계입니다.

### 16-1. 1단계 — 후보 추리기 (아래 **전부** 통과해야 추천에 등장)
1. 병원 **계약 상태 = active** (아니면 탈락)
2. **그 시술이 "현재 제공 중"** 체크돼 있어야 함
3. 가격 공개 병원이면 **시작 가격 ≤ 환자 예산**
4. **시술 다운타임 ≤ 환자가 허용한 회복일**
5. **시술 통증 ≤ 환자가 허용한 통증**
6. **그 시술이 환자가 고른 고민 중 최소 1개와 연결**돼 있어야 함 (= 고민↔시술 매트릭스)

→ 6번 때문에 매트릭스가 비어있으면 **아무도 추천되지 않습니다.**

### 16-2. 2단계 — 점수 매겨 줄 세우기 (높은 순서로 최대 8곳)
| 신호 | 점수 | 운영자가 조절하는 곳 |
|---|---|---|
| 고민 관련도 | primary +50 / secondary +28 / adjunct +12 | 고민↔시술 매트릭스의 "관련도" |
| 스타일 적합 | 최대 +24 | 시술의 "강도" |
| 예산 여유 | 최대 +12 | 병원×시술 "시작 가격" |
| 할인율 | 최대 약 +25 | 병원×시술 가격/이벤트 |
| 시그너처 | +10 | 병원×시술 "병원 시그너처" 체크 |
| 진행 이벤트 | +6 | 병원×시술 "이벤트 진행 중" |
| 언어 지원 | +8 | 병원 "응대 가능 언어" |
| 영어 의사 | +3 | 병원 "영어 가능 의사" |
| 공항 픽업 | +3 | 병원 "공항 픽업" |
| 국제 코디 | +3 | 병원 "국제 코디네이터" |
| 선호 기기 일치 | 병원이 실제 사용 +18 / 시술-기기 매트릭스 primary +9 / alternative +5 | 병원×시술 "사용 장비" + 시술↔기기 매트릭스 |

- **고민 관련도(+50)가 가장 셉니다.** 그래서 매트릭스가 추천 품질의 80%입니다.
- 같은 시술을 여러 병원이 하면 **그중 점수 1등 병원만** 결과에 노출됩니다(같은 시술 도배 방지).

## 17. 자주 빠지는 함정 (체크리스트)

1. **병원이 사이트에 안 보임** → 십중팔구 **계약 상태가 active 가 아님.** (1순위 점검)
2. **추천에 아무 병원도 안 나옴** → 고민↔시술 매트릭스가 비었거나, 그 시술이 환자 고민과 연결 안 됨.
3. **가격이 "상담"으로만 나옴** → 병원×시술의 **"가격 공개 허용"** 체크 안 함.
4. **URL 식별자를 나중에 바꿈** → 링크·사진 다 깨짐. **절대 바꾸지 마세요.**
5. **전후사진 동의 없이 등록** → 개인정보보호법 위반. "서면 동의 보유" 없으면 등록 금지.
6. **별표(*) 칸 비움** → 저장 안 됨.
7. **사진 비움** → 카드가 회색 빈 박스. 최소한 썸네일은 넣으세요.
8. **순서 어김** → 시술 없이 병원×시술을 못 만들고, 고민·시술 없이 매트릭스를 못 만듭니다. §1 순서대로.

## 18. 한눈 요약 — "이건 어디서 관리하나"

| 하고 싶은 것 | 어디서 |
|---|---|
| 새 병원 등록 | 병원 메뉴 → + 새로 만들기 (브랜드·시술·의사·전후사진 다 그 안에서) |
